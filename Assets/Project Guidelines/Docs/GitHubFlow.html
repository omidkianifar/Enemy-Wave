GitHub Flow Command Summary Sheet

1. Initial Setup

Create a GitHub Repository

Create a new repository on GitHub through the website. Initialize with a README (optional).

Clone the Repository

git clone https://github.com/yourusername/YourRepository.git
cd YourRepository

Clones the GitHub repository to your local machine.

Initialize Git LFS

git lfs install

Sets up Git LFS in your repository to handle large files.

Create a .gitignore File

curl https://raw.githubusercontent.com/github/gitignore/main/Unity.gitignore -o .gitignore

Downloads a Unity-specific .gitignore file to ignore unnecessary files in the repository.

Commit .gitignore and Set Up Git LFS

git add .gitignore
git commit -m "Add Unity .gitignore file"

git lfs track "*.psd"
git lfs track "*.png"
git lfs track "*.fbx"
git lfs track "*.wav"
git lfs track "*.mp4"

git add .gitattributes
git commit -m "Set up Git LFS for large files"

Push Initial Commit

git push origin main

Pushes the initial commit to GitHub.

2. Feature Development

Create a Feature Branch

git checkout -b feature/feature-name

Creates a new branch for a feature.

Work and Commit

git add .
git commit -m "Describe your changes"

Adds and commits changes to the feature branch.

Push the Feature Branch

git push origin feature/feature-name

Pushes the feature branch to GitHub.

Create a Pull Request

On GitHub, create a PR from your feature branch to main. This is where code review happens.

Merge the PR and Delete the Branch

git checkout main
git pull origin main
git branch -d feature/feature-name

Merges the feature branch and deletes it locally.

3. Bug Fixes

Create a Bugfix Branch

git checkout -b bugfix/fix-description

Creates a branch for the bug fix.

Work and Commit

git add .
git commit -m "Fix description"

Push the Bugfix Branch

git push origin bugfix/fix-description

Pushes the bugfix branch to GitHub.

Merge the PR and Delete the Branch

git checkout main
git pull origin main
git branch -d bugfix/fix-description

4. Hotfixes

Create a Hotfix Branch

git checkout -b hotfix/fix-critical-issue

Creates a branch for a critical hotfix.

Work and Commit

git add .
git commit -m "Critical issue fix"

Push the Hotfix Branch

git push origin hotfix/fix-critical-issue

Merge the PR and Delete the Branch

git checkout main
git pull origin main
git branch -d hotfix/fix-critical-issue

5. Releases

(Optional) Create a Release Branch

git checkout -b release/vX.X

Creates a branch for finalizing a release.

Finalize and Test

git add .
git commit -m "Final touches for release"

Merge the Release Branch

git checkout main
git merge release/vX.X

Tag the Release

git tag -a vX.X -m "Release version X.X"
git push origin vX.X

Push and Delete the Release Branch

git push origin main
git branch -d release/vX.X

6. General Maintenance

Regularly Pull Updates

git checkout main
git pull origin main

Keeps your local main branch up to date.

List Tracked LFS Files

git lfs ls-files

Shows the files currently tracked by Git LFS.
